Basic Structure:

  • Each HTML document can be considered a "Document Tree".
  • HTML is organized into a family tree structure. Such as grand-parents, parents, children, siblings etc.
  • Every HTML Document has the following in order: !DOCTYPE, html (opening & closing),head(O&C) & body (O&C) inside html.
  • Any visible content should be placed within the opening and closing body tags.
  • Headings and sub-headings, h1 to h6 tags, are used to provide titles for sections of content.
  • You can't wrap a block element with an inline element. Difference between the two.

Nesting:

  • HTML elements can be nested inside other elements. The enclosed element is the child of the enclosing parent element.
  • The relationship between elements and their ancestor and descendent elements is known as hierarchy.
  • Children are nested within parent are shown via a tab/2spaces of indentation for better readability.
    1. DOCTYPE and html are same nesting level,
    2. Head and body are the children of HTML but siblings to each other. They are on indented under head, but same nesting to each other.
    3. Elements such as main & footer go within body as a child element.
    4. Elements like div & footer can be both parents and children depending on order/structure.
  • There can be multiple levels of nesting thus extending to grand and great grand children.
  • SELF REMINDER: elements within elements are children, text within element does NOT need to be formatted as a child.
  • SELF REMINDER: you don't need to add a space after a tag to write a sentence. YOu can just type immediatley after the >.
