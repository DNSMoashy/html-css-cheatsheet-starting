Basic Semantic Tags & Definitions

  • Article:Represents a part of a page which is self-contained and could be published elsewhere. Common uses include blog posts or magazine articles.
  • Aside:Represents a part of a page which is relevant to the current content, but not a part of it directly. It can be used for related links, for clarifying a statement from the current article, or even for advertising meant for the current page.
  • Details:Encapsulates the summary element and any additional details which are only visible when the element is in an open state.
  • Fig Caption:Describes the media encapsulated within the figure element.
  • Figure:Encapsulates media such as an image, diagram, or code snippet.
  • Footer:Represents a part of a page which is meant to be at the end of a completed block of content.Common uses include copyright information for the page or additional links to relevant pages.
  • Header:Represents a part of a page which is meant to be introductory. It can include heading tags, a logo, a search bar, and navigation elements.
  • Main:Represents the primary content within the body element of the web page.
  • Mark:Represents part of a text that should be rendered as marked or highlighted.
  • Nav:Defines a block of navigation links such as menus and tables of contents.
  • Section:Defines elements in a document, such as chapters, headings, or any other area of the document with the same theme.
  • Summary:Acts as a label for the details element and reveals additional information when clicked.
  • Time:Represents a time-related piece of information and can take a datetime attribute that makes it machine-readable.

< !DOCTYPE html>, < html >, DOC-yes, html-no

Required bones of every HTML document.

  • DOCTYPE declares what type of document (ie. HTML)
  • HTML (the root) is what goes after & Contains ALL your code
  • is NOT an element or tag.
< head > < /body > no
  • HEAD contains metadata (general info about the page) like < title > or stylesheet links that are not shown on page itself.
  • BODY is the content that is displayed on the browser and will contain all your elements and coding.
< div > no

"division" An element that is used to group sections. Can contain other elements & "flow content".

< section > no

Similar to div. Defines elements such as chapters or areas w/ same theme. More semantic then it but less semantic than the following ...

Can be used with Article

< span > no

Inline text container used for group styling in CSS. Is generic so use more semantic elements if possible.

< header >, < nav >, < main >, < footer > no

The Semantic versions of div tag. Goes within the body tag.

< em > & < strong > no

Inline styling used to emphasize. em will italicize & strong will make bold.

< a > no

Anchor element used to create hyperlink via href (location).

Can link to other pages, files, or parts within same page. Use with Target Attribute.

< video >, & < audio > no

Allows us to add video and audio to our site. Universal elements that use the src to link content.

< embed > yes

Used to implement any type of media. Is self-closing.

< img > yes

embeds image into document.

< figure > and < figcaption > no

Figure holds media within it, while Fig Caption will describe it. Together these elements group media & description.

< article > no

Holds content that can stand alone.

Sometimes used with Section (which defines a theme) to write stand alone content for said theme.

< aside > no

Adds additional info that enhances content. Not required to understand main content

Similar to a side bar/comment related to content but not obstructing it.

< ol > & < ul > no

Ordered ( #'s) & Unordered ( Bullet point's) lists.

Must contain < li > < /li > within both.

< h1 - h6 >, < p > no

Heading values from largest (h1) to smallest (h6)

P contains your blocks of text.
